# Лабораторная работа №6
### Вариант 10: Расчёт расхода ткани и стоимости пошива одежды

---


### Задача
Разработать программу для автоматизации расчёта расхода ткани и стоимости пошива одежды с использованием модулей и пакетов в Python.

| № | Вид одежды | Особенности |
|---|------------|-------------|
| 1 | **Пиджак** | Расход ткани: 2.2-2.7 м в зависимости от размера |
| 2 | **Брюки** | Расход ткани: 1.2-1.45 м в зависимости от размера |
| 3 | **Костюм-тройка** | Пиджак + Брюки + Жилет (0.8 м ткани) |

### Исходные данные для расчётов

**Таблица расхода ткани по размерам:**

| Размер | Пиджак (м) | Брюки (м) | Жилет (м) |
|--------|------------|-----------|-----------|
| 44 | 2.2 | 1.2 | 0.8 |
| 46 | 2.3 | 1.25 | 0.8 |
| 48 | 2.4 | 1.3 | 0.8 |
| 50 | 2.5 | 1.35 | 0.8 |
| 52 | 2.6 | 1.4 | 0.8 |
| 54 | 2.7 | 1.45 | 0.8 |

**Финансовые параметры:**

| Параметр | Пиджак | Брюки | Жилет |
|----------|--------|-------|-------|
| Цена ткани за 1 метр | 800 руб | 800 руб | 800 руб |
| Стоимость работы за 1 метр | 1200 руб | 1000 руб | 1000 руб |
| Стоимость фурнитуры | 500 руб | 300 руб | 200 руб |

### Реализация пакета и модулей

#### Файл `__init__.py`

```python
from .coat import CoatCalculator
from .trousers import TrousersCalculator
from .suit import SuitCalculator

__all__ = ['CoatCalculator', 'TrousersCalculator', 'SuitCalculator']
```
<br>

- Инициализирует пакет clothing_package, позволяя Python распознавать папку как полноценный пакет для импорта
- Импортирует все три класса-калькулятора (CoatCalculator, TrousersCalculator, SuitCalculator) из соответствующих модулей
- Определяет __all__ — список публичных объектов, которые доступны при импорте from clothing_package import *
- Обеспечивает удобный импорт: вместо from clothing_package.coat import CoatCalculator можно писать from clothing_package import CoatCalculator

<br>

#### Для файла coat.py (Модуль для расчёта пиджака)
- Хранит справочные данные: таблицу расхода ткани для размеров 44-54, цены на материалы и работу
- Содержит метод calculate_fabric() — определяет нужное количество ткани с автоподбором ближайшего размера, если введён нестандартный (например, 45 → 46)
- Содержит метод calculate_price() — вычисляет полную стоимость пиджака по формуле:
(расход ткани × цена ткани) + (расход ткани × стоимость работы) + стоимость фурнитуры
- Возвращает словарь с детальной разбивкой: тип одежды, размер, расход ткани, стоимость ткани, работы, фурнитуры и итоговую сумму

<br>

#### Для файла trousers.py (Модуль для расчёта брюк)

- Хранит нормы расхода ткани для брюк (1.2–1.45 м в зависимости от размера)
- Содержит метод calculate_fabric() — определяет расход ткани с округлением до ближайшего размера
- Содержит метод calculate_price() — вычисляет стоимость брюк по аналогичной формуле, но с другими параметрами (меньший расход ткани, дешевле работа и фурнитура)
- Возвращает структурированный словарь с результатами расчёта

<br>

#### Для файла suit.py (Модуль для расчёта костюма-тройки)

- Импортирует два предыдущих калькулятора (CoatCalculator и TrousersCalculator) для переиспользования их логики
- Добавляет расчёт жилета: фиксированный расход 0.8 м ткани, своя стоимость работы и фурнитуры
- Метод calculate_price():
  - Создаёт объекты калькуляторов пиджака и брюк
  - Получает результаты расчёта для пиджака и брюк
  - Рассчитывает стоимость жилета
  - Суммирует все три стоимости в общую цену костюма
- Возвращает сложный словарь, содержащий вложенные результаты для каждой детали костюма

<br>

#### Для файла main.py

- Реализует графический интерфейс пользователя на базе библиотеки tkinter
- Обеспечивает взаимодействие пользователя с расчётными модулями
- Предоставляет возможность сохранения результатов в форматах .docx и .xlsx

<br>

#### Для файла requirements.txt
```
python-docx==1.1.0
openpyxl==3.1.2
```
- Перечисляет внешние зависимости, необходимые для работы программы
- python-docx — библиотека для создания и редактирования документов Microsoft Word (.docx)
- openpyxl — библиотека для работы с файлами Excel (.xlsx)
- Фиксация конкретных версий гарантирует совместимость и устраняет «эффект неожиданности» при обновлениях библиотек
- Позволяет установить все зависимости командой pip install -r requirements.txt
